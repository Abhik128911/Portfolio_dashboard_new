S-08 / S-51 / L-07 Define Type I and Type II errors with practical examples ▼
Answer
Type I Error (False Positive)
  • Definition: Rejecting a TRUE null hypothesis
  • In Statistics: Concluding an effect exists when it doesn't
  • ML Example 1: Classifying a genuine email as spam
  • ML Example 2: A healthy patient diagnosed as diseased
  • Symbol: α (significance level)
  • Also called: False Alarm
Type II Error (False Negative)
  • Definition: Failing to reject a FALSE null hypothesis
  • In Statistics: Missing a real effect
  • ML Example 1: Spam email classified as normal (missed)
  • ML Example 2: A diseased patient diagnosed as healthy
  • Symbol: β
  • Also called: Miss
Key Tradeoff
Reducing Type I errors (lower α) tends to increase Type II errors and vice versa. The balance depends on application — in medical diagnosis, Type II errors can be life-threatening, so high recall (low Type II) is prioritized.
S-04 / L-05 State and describe the Central Limit Theorem (CLT) and its significance in ML ▼
Answer
CLT: As sample size n → ∞, X̄ ~ N(μ, σ²/n) regardless of population distribution (n ≥ 30 typically sufficient)

Statement: The sampling distribution of the sample mean approaches a normal (Gaussian) distribution as the sample size increases, regardless of the population's shape.

Key Points:

  • Requires independent observations and finite variance
  • Works even if the underlying population is skewed or uniform
  • The approximation improves as n increases
Significance in Machine Learning
  • Algorithm Assumptions: Many ML models (Linear Regression, Naïve Bayes) assume normally distributed errors — justified by CLT
  • Hypothesis Testing: Enables statistical inference on model performance
  • Confidence Intervals: Helps estimate population parameters from sample data
  • Bootstrapping & Cross-Validation: Relies on CLT for stability
  • Feature Scaling: Z-score normalization is meaningful due to CLT

S-10 / S-54 Significance of PDF (Probability Density Function) in ML. Calculate PDF for μ=60, σ²=7, x=65 ▼
Answer

A Probability Density Function (PDF) describes the probability distribution of a continuous random variable. Unlike discrete distributions, it gives the relative likelihood of a value occurring in a range (probability = area under the curve).

Key roles in ML:

  • Probability estimation and statistical modeling
  • Kernel Density Estimation (KDE) for non-parametric modeling
  • Generative models (VAEs, GANs) rely on learned PDFs
  • Anomaly detection — points in low-probability regions are outliers
Given: μ = 60, σ² = 7, x = 65 → σ = √7 ≈ 2.6458

f(x) = (1/(σ√2π)) · e^(−(x−μ)²/(2σ²))
= (1/(2.6458·√2π)) · e^(−(65−60)²/(2·7))
= (1/6.6228) · e^(−25/14)
= 0.1510 · e^(−1.7857)
= 0.1510 · 0.1674
f(65) ≈ 0.025
S-12 / L-02 / L-03 Sampling Distribution, Binomial Distribution, population & sample, sampling techniques ▼
Answer

Population: The entire set of data points under study. In ML, it represents the complete dataset from which models are derived.

Sample: A representative subset of the population, used to reduce computational cost while maintaining accuracy.

Sampling Distribution: The probability distribution of a statistic (e.g., mean) obtained from multiple samples. It helps estimate population parameters.

Binomial: P(X=k) = C(n,k)·pᵏ·(1−p)^(n−k)  |  k successes in n trials, each with probability p

S-13 Define permutations and combinations with examples in the context of ML ▼
Answer
Permutation: P(n,r) = n!/(n−r)!     Combination: C(n,r) = n! / (r! · (n−r)!)
Permutation (Order Matters)
  • Ordered arrangement of r items from n
  • Example: 3 hyperparameters in 3! = 6 orderings for tuning
  • Used in: hyperparameter search, sequence modeling
Combination (Order Doesn't Matter)
  • Selection of r items from n, without regard to order
  • Example: Choose 3 features from 10: C(10,3) = 120 subsets
  • Used in: feature selection, ensemble methods

S-07 Define the rank of a matrix and its importance in ML ▼
Answer

The rank of a matrix is the number of linearly independent rows or columns in it. It represents the dimensionality of the vector space spanned by those rows/columns.

rank(A) = max(number of linearly independent rows, number of independent columns)

If rank(A) = min(m, n) → Full rank (non-singular) — unique solution exists
If rank(A) < min(m, n) → Rank-deficient (singular) — infinite or no solutions

Applications in ML
  • Solving linear equations in regression (checking if A has full rank)
  • PCA: dimensionality = effective rank of covariance matrix
  • SVD: rank equals number of non-zero singular values
  • Feature engineering: detecting redundant/collinear features
S-14 / L-09 Importance of Eigenvalues & Eigenvectors in ML; define Vectors & Matrices ▼
Answer
Eigenvector equation: A·v = λ·v   |   Eigenvalue equation: det(A − λI) = 0

Eigenvector (v): A non-zero vector whose direction does not change under a linear transformation A. Only its magnitude may scale.

Eigenvalue (λ): The scalar that indicates how much the eigenvector is scaled during the transformation.

Vectors
  • 1D array: magnitude + direction
  • Represent features or data points
  • Row vector [x₁,x₂,...xₙ] or column
  • Used in: model parameters, embeddings
Matrices
  • 2D array: rows × columns
  • Store entire datasets (rows=samples)
  • Weight matrices in neural networks
  • Used in: transformations, covariance
Applications of Eigenvalues/Eigenvectors
  • PCA: Eigenvectors of covariance matrix = principal components; eigenvalues = variance explained
  • Spectral Clustering: Uses eigendecomposition of graph Laplacian
  • Eigenfaces: Facial recognition using eigenvectors of pixel covariance
  • Stability Analysis: Eigenvalues of Hessian matrix guide gradient descent
  • PageRank: Dominant eigenvector of web graph

S-18 / S-20 / S-21 / L-13 / L-14 / L-15 Supervised Learning — definition, labeled data, training/test split, types, steps, methodology ▼
Answer

Supervised Learning trains a model on labeled data (input–output pairs) to learn a mapping function f(X) → y, then makes predictions on new unseen data.

Classification
  • Output: discrete class labels
  • Binary: spam/not spam
  • Multi-class: digit recognition
  • Algorithms: Decision Tree, SVM, Logistic Regression, KNN, Naïve Bayes
Regression
  • Output: continuous numerical values
  • Examples: house price, stock price
  • Algorithms: Linear Regression, Polynomial Regression, Ridge, Lasso
  • Metric: MSE, RMSE, R²

Steps in Supervised Learning:

  1. Data Collection — Gather labeled input-output pairs
  2. Preprocessing — Handle missing values, normalize, encode categories
  3. Train-Test Split — Typically 70-80% train, 20-30% test
  4. Algorithm Selection — Choose based on problem type and data characteristics
  5. Model Training — Optimize parameters to minimize error (e.g., gradient descent)
  6. Evaluation — Test on unseen data using accuracy, precision, recall, F1, RMSE
  7. Hyperparameter Tuning — Grid search / random search to optimize
  8. Deployment — Serve model in production; monitor and retrain as needed
Labeled Data
Labeled data consists of input features paired with correct output labels. Example: emails labeled "spam" or "not spam". Without labels, learning direction is impossible in supervised settings. Manual labeling is expensive but tools like active learning and semi-supervised methods reduce the burden.

L-17 Illustrate the seven major steps involved in the Machine Learning Life Cycle ▼
Answer
  1. Problem Definition — Clearly state the objective; decide supervised/unsupervised/RL; define success metrics
  2. Data Collection — Gather from databases, APIs, web scraping, sensors; ensure relevance and diversity
  3. Data Preprocessing — Handle missing values, normalize, encode categoricals, remove duplicates & outliers
  4. Model Selection — Choose algorithm based on problem type, data size, interpretability needs
  5. Model Training — Feed training data; optimize parameters via gradient descent or other optimizers
  6. Model Evaluation — Test on unseen data; measure accuracy, F1, RMSE etc.; use cross-validation
  7. Model Deployment — Integrate into production via API/cloud; monitor performance; retrain as data drifts
Note
The life cycle is iterative — poor evaluation results trigger going back to data collection, feature engineering, or model selection. Continuous monitoring ensures long-term effectiveness.

L-30 Significance of Dimensionality Reduction — PCA explained ▼
Answer

PCA (Principal Component Analysis) finds a new orthogonal coordinate system where axes (principal components) align with directions of maximum variance in the data.

  1. Standardize — Zero mean, unit variance
  2. Covariance Matrix — Compute Σ = XᵀX/(n−1)
  3. Eigendecomposition — Get eigenvalues (λ) and eigenvectors (v) of Σ
  4. Sort — Sort eigenvectors by eigenvalues descending
  5. Project — X' = X · V_k (top-k eigenvectors)
Benefits
Reduces dimensionality → faster training, less overfitting, easier visualization. Removes multicollinearity. Retains maximum variance with fewest dimensions.
Limitation
Principal components lose interpretability as they are linear combinations of original features. PCA assumes linear relationships.

S-38 / S-39 Why is Train-Test split essential? Explain model selection importance. ▼
Answer

Train-Test Split partitions data so the model trains on one portion and is evaluated on a completely unseen portion, mimicking real-world deployment.

Common splits: 70/30 · 80/20 · 60/20/20 (Train/Validation/Test)

Without it: the model memorizes training data (overfitting) and we have no unbiased estimate of generalization.

Model Selection involves choosing the best algorithm for the data based on:

  • Dataset characteristics (size, dimensionality, noise)
  • Problem type (classification/regression/clustering)
  • Computational constraints
  • Interpretability requirements
  • Cross-validation performance across multiple models

Linear Regression
S-23 / L-40 / L-41 / L-43 Linear Regression — concept, equation, cost function, gradient descent, simple vs multiple ▼
Answer
Simple: y = β₀ + β₁x + ε   |   Multiple: y = β₀ + β₁x₁ + β₂x₂ + ... + βₙxₙ + ε
Matrix form: y = Xβ + ε  →  β = (XᵀX)⁻¹Xᵀy (Ordinary Least Squares)

Cost Function (MSE):

J(β) = (1/n) Σ(yᵢ − ŷᵢ)² = (1/n) Σ(yᵢ − β₀ − β₁xᵢ)²

Gradient Descent: Iteratively updates parameters to minimize J(β):

βⱼ := βⱼ − α · ∂J/∂βⱼ    (α = learning rate)
AspectSimple Linear RegressionMultiple Linear Regression
PredictorsOne independent variableMultiple independent variables
ExampleHouse price ~ Area onlyHouse price ~ Area + Bedrooms + Location
InterpretationEasier, single slopeMore complex; each β is partial effect
AccuracyLimited — ignores other factorsHigher — captures combined effect
Advantages & Limitations
✓ Simple, interpretable, fast training    ✗ Assumes linearity, sensitive to outliers, multicollinearity issues

Pre-Pruning (Early Stopping)
  • Stop tree growth based on conditions (max depth, min gain)
  • Faster, avoids useless branches
  • Risk: might stop too early (underfitting)
Post-Pruning
  • Allow full growth, then trim branches
  • Methods: Cost Complexity Pruning (α)
  • Better generalization; more thorough
Decision Trees vs Logistic Regression
Decision trees handle non-linear boundaries, are highly interpretable but prone to overfitting. Logistic regression is linear, stable, better generalization on linear problems. Use DT when complex patterns exist; LR for simpler interpretable models.
K-Means & Hierarchical Clustering
S-47 / L-20 / L-45 / L-49 K-Means Clustering — algorithm, example, vs Hierarchical Clustering, when to use which ▼
Answer

K-Means Algorithm Steps:

  1. Initialize — Randomly place K centroids in feature space
  2. Assign — Assign each data point to nearest centroid (by Euclidean distance)
  3. Update — Recalculate centroids as mean of assigned points
  4. Repeat — Repeat assign+update until centroids stop moving (convergence)
Objective: minimize J = Σₖ Σᵢ∈Cₖ ||xᵢ − μₖ||²  (within-cluster sum of squares)

Choosing K: Use the Elbow Method — plot inertia vs K; pick K at the "elbow" point.

ApproachPartition-based (flat)Tree-based (dendrogram)
K required?Yes, must specify K upfrontNo — cut dendrogram at desired level
ComplexityO(n·K·I) — scalableO(n²·log n) — expensive for large data
OutputK flat clustersHierarchy / dendrogram
Best forLarge datasets, globular clustersSmall datasets, exploring hierarchical structure
Outlier sensitivityHigh (centroids pulled by outliers)Moderate
Distance metricEuclidean by defaultEuclidean, Manhattan, Cosine (flexible)
Real-World Example: Customer Segmentation
An e-commerce platform uses K-Means (K=4) on: purchase frequency, average spend, product categories. Segments: Budget Shoppers · Loyal Customers · Occasional Buyers · High Spenders. Each cluster gets tailored marketing campaigns.
K-Means Limitations
Assumes spherical clusters · Sensitive to outliers · Must specify K in advance · May converge to local minima (use K-Means++ initialization)

S-43 / S-52 / L-34 / L-51 Confusion Matrix — structure, all metrics derived from it, Sensitivity vs Specificity ▼
Answer
Predicted: Positive
Predicted: Negative
Actual: Positive
TP

Actual: Negative
FP

Binary Classification Confusion Matrix
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)     [How many predicted positives are correct?]
Recall (Sensitivity) = TP / (TP + FN)     [How many actual positives were found?]
Specificity (TNR) = TN / (TN + FP)     [How many actual negatives were correctly rejected?]
F1-Score = 2 × (Precision × Recall) / (Precision + Recall)
FPR = FP / (FP + TN)     FNR = FN / (FN + TP)

Sensitivity vs Specificity Tradeoff
Increasing sensitivity (lower threshold) → catches more true positives → but more false positives (lower specificity). This is the ROC curve tradeoff — choose operating point based on application consequences.
